MAX6675 K-Type Thermocouple Temperature Sensor¶

The max6675 temperature sensor allows you to use your max6675 thermocouple temperature sensor (datasheet, sainsmart) with esphomelib

MAX6675 K-Type Thermocouple Temperature Sensor.

As the communication with the MAX66775 is done using SPI, you need to have an spi bus in your configuration with the miso_pin set (mosi is not required).

Connect GND to GND, VCC to 3.3V and the other three MISO (or SO for short), CS and CLOCK (or CLK) to free GPIO pins.

# Example configuration entry
spi:
  miso_pin: D0
  clk_pin: D1

sensor:
  - platform: max6675
    name: "Living Room Temperature"
    cs_pin: D2
    update_interval: 60s

Configuration variables:¶

  • name (Required, string): The name for the temperature sensor.
  • cs_pin (Required, Pin Schema): The Chip Select pin of the SPI interface.
  • update_interval (Optional, Time): The interval to check the sensor. Defaults to 60s.
  • spi_id (Optional, ID): Manually specify the ID of the SPI Component if you want to use multiple SPI buses.
  • id (Optional, ID): Manually specify the ID used for code generation.
  • All other options from Sensor and MQTT Component.
